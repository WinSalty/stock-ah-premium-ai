# 移动端 App 化体验改造方案

文档日期：2026-05-18

## 1. 背景与目标

当前前端以桌面投研工作台为主，移动端主要依赖响应式换行和侧栏压缩。智能问答页面虽然已经具备会话、流式回答、数据摘要、导出和雪球草稿能力，但手机首屏会被会话列表、页面标题和后台式侧边导航占用，用户难以把问答当作主入口使用。

本轮改造目标是在不改变桌面端页面结构和主视觉的前提下，为移动端提供类似原生 App 的 Web 体验：

- 移动端登录后优先进入“问答”主界面；若用户没有问答权限，则进入其第一个可访问页面。
- 移动端使用顶部应用栏、底部主导航和更多菜单承载全局入口，不再展示桌面左侧侧边栏。
- 智能问答在移动端采用聊天 App 结构：会话列表改为抽屉，聊天历史占据主区域，输入区固定在底部。
- 保持现有 API、权限、会话、流式回答、导出、雪球草稿和数据摘要能力不变。

## 2. 风险点记录

### 2.1 桌面端回归风险

桌面端当前布局由 `App.tsx` 的 `Layout.Sider` 和各页面既有样式共同承担。若直接改全局 `.page`、`.app-shell` 或聊天布局，可能影响桌面端投研工作台。因此本轮必须通过移动端专用应用壳和 `max-width: 720px` 样式隔离，避免改变桌面端 DOM 主路径和宽屏 CSS。

### 2.2 权限与默认首页风险

普通用户、管理员和自定义授权用户的菜单权限不同。移动端默认进入问答时必须先判断 `chat` 权限；没有问答权限时，只能回退到当前权限列表的第一个页面，避免出现空白页或越权入口。

### 2.3 软键盘与视口高度风险

移动端浏览器特别是 iOS Safari 会在键盘弹起时改变可视高度。聊天历史、输入框和底部导航需要使用 `100dvh`、`env(safe-area-inset-bottom)` 和内部滚动区，避免输入框被遮挡、聊天历史无法滚动或页面整体双重滚动。

### 2.4 流式回答滚动风险

问答页已有“贴近底部才自动跟随”的流式滚动逻辑。移动端输入区固定后，聊天历史高度变化更频繁，必须保留既有滚动判断，不能为了 app 化布局重新强制滚到底部，否则用户阅读长回答时会被打断。

### 2.5 数据摘要与 Markdown 表格风险

LLM 回答可能包含宽表格和数据摘要。手机上不能压缩所有列，否则会破坏事实核验；也不能让表格撑破页面。因此移动端继续保留横向滚动容器，并缩小边距、字号和卡片内留白。

### 2.6 低频操作拥挤风险

导出、发布雪球、批量删除、模型选择、新建会话都在问答页存在。移动端若全部铺开，会挤压主要问答区域。本轮将高频操作保留在主界面，将会话管理放入抽屉，将全局低频页面放入更多菜单。

### 2.7 触摸命中与误删风险

会话删除、批量删除属于高影响操作。移动端抽屉中继续使用确认弹窗，按钮维持足够触摸尺寸，避免滑动列表时误触删除。

### 2.8 验证覆盖风险

本轮主要是前端体验重构，必须至少执行前端构建，并通过本地浏览器分别查看桌面和移动视口。若本机服务或真实登录态不可用，应在交付说明中明确验证缺口。

## 3. 开发计划

### 3.1 应用壳改造

1. 在 `App.tsx` 增加移动端视口识别逻辑。
2. 保留桌面端现有 `Layout.Sider + Layout.Content` 主路径。
3. 新增移动端应用壳：顶部应用栏、底部主导航、更多菜单抽屉和用户退出区域。
4. 移动端主导航优先展示“问答、AH 机会筛选、总览、查询、个人信息”，其余授权页面进入更多菜单。
5. 登录成功和登录态恢复后，移动端优先选择 `chat` 页面；没有权限则回退到第一个授权页面。

### 3.2 智能问答移动端改造

1. 在 `ChatPage.tsx` 中复用现有会话列表逻辑，提取为可同时放入桌面侧栏和移动端抽屉的渲染片段。
2. 移动端新增问答内部顶部栏，提供“会话抽屉、当前会话标题、新建会话”三个高频入口。
3. 移动端隐藏桌面会话侧栏和桌面问答工具栏，避免重复占用首屏。
4. 聊天历史保留内部滚动，输入区固定在问答页面底部。
5. 保留模型选择、发送、单条导出、当前会话导出和雪球草稿能力。

### 3.3 样式与响应式隔离

1. 新增 `.mobile-app-shell`、`.mobile-app-header`、`.mobile-app-content` 和 `.mobile-app-tabbar` 样式。
2. 在移动端应用壳下覆盖 `.page`、`.chat-page`、`.chat-workspace`、`.chat-main`、`.chat-history` 和 `.chat-composer`。
3. 保持桌面端 `.app-shell`、`.app-sider`、`.chat-workspace` 默认样式不变。
4. 对问答气泡、回答卡片、数据摘要、Markdown 表格和预设问题按钮做手机尺寸适配。

### 3.4 验收计划

1. 执行 `npm --prefix frontend run build`。
2. 启动前端本地服务，使用浏览器检查桌面视口，确认侧边栏和桌面问答布局仍可用。
3. 使用移动视口检查登录后默认问答、底部导航、更多菜单、会话抽屉、输入区和长回答滚动。
4. 若无可用登录态或后端服务，至少完成静态页面壳、构建和可访问性结构检查，并记录未完成的真实接口验证。

## 4. 交付边界

本轮只改前端应用壳、智能问答移动端布局和正式项目文档，不改后端接口、数据库结构、权限定义和 LLM 业务策略。桌面端页面应保持原路径、原组件和原视觉结构。
