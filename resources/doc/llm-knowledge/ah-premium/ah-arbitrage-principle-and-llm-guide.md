# A/H 股价差套利原理与 LLM 推荐指南

更新日期：2026-05-04

## 1. 文档目标

本文说明 A/H 股价差套利的核心原理、可执行边界、风险点和产品中的 LLM 推荐方式，并基于本地已同步的 Tushare 官方 AH 比价数据给出实际观察例子。

A/H 价差长期存在，不代表一定会收敛；真实交易还要考虑账户权限、融券可得性、港币汇率、交易成本、税费、流动性、卖空限制和资金占用。

## 2. 基础概念

A/H 股指同一家公司同时在内地 A 股市场和香港 H 股市场上市。两地股票代表同一家公司经济权益，但交易币种、投资者结构、资金成本、流动性、卖空机制和监管制度不同，所以价格经常不一致。

项目当前使用 Tushare `stk_ah_comparison` 官方 AH 比价接口作为主口径，字段包括 A 股代码、H 股代码、交易日、A/H 比价和 A/H 溢价。官方文档见 [Tushare AH 股比价](https://tushare.pro/document/2?doc_id=399)。

互联互通背景可参考：

- [HKEX Stock Connect](https://www.hkex.com.hk/Mutual-Market/Connect-Hub/Stock-Connect?sc_lang=en)：沪深港通提供香港与内地市场之间的互联互通交易安排。
- [上海证券交易所 Stock Connect introduction](https://english.sse.com.cn/access/stockconnect/introduction/)：说明北向和南向合资格证券交易机制。
- [证监会沪港通问答](https://www.csrc.gov.cn/csrc/c100210/c1498896/content.shtml)：说明沪港通下港股通股票范围、禁止裸卖空等规则边界。

## 3. 价差计算口径

项目视图中使用两个方向：

```text
AH 溢价 = A 股人民币价格 / H 股人民币折算价格 - 1
H/A 溢价 = H 股人民币折算价格 / A 股人民币价格 - 1
```

解释：

- `AH 溢价 > 0`：A 股相对 H 股更贵，H 股相对更便宜。
- `AH 溢价 < 0`：A 股相对 H 股更便宜，H 股相对更贵。
- `H/A 溢价` 是反方向观察，适合已经持有 A 股或 H 股时判断替代和轮动。

示例：如果某公司 A 股折算后为 110 元，H 股折算后为 100 元，则 AH 溢价约为 10%。这并不表示可以无风险赚 10%，因为 A 股和 H 股不能简单跨市场转托管互换，做空和融资成本也不同。

## 4. 套利与替代交易的区别

严格套利通常需要同时完成：

1. 买入低估一侧。
2. 卖空或对冲高估一侧。
3. 等待价差收敛后同时平仓。

但 A/H 场景中，普通投资者常见限制包括：

- A 股和 H 股通常不能直接互换交割。
- 港股通和沪深股通只覆盖合资格证券。
- 裸卖空受限制，融券券源、成本和期限不确定。
- 两地交易日、交易时段、涨跌停规则和结算制度不同。
- 汇率波动会影响折算后的价差。

所以产品里更适合把它称为“价差观察、跨市场替代、组合再平衡机会”，而不是承诺无风险套利。

## 5. 常见交易思路

### 5.1 A 贵 H 便宜

当 `AH 溢价` 显著为正时，代表 A 股更贵，H 股更便宜。

可能动作：

- 已持有 A 股：考虑是否切换到 H 股，降低买入估值。
- 新建仓：优先观察 H 股一侧。
- 专业投资者：若能合法做空 A 股或用指数/行业工具对冲，可考虑多 H 空 A 的价差组合。

### 5.2 A 便宜 H 贵

当 `AH 溢价` 为负时，代表 A 股更便宜，H 股更贵。

可能动作：

- 已持有 H 股：观察是否切换到 A 股。
- 新建仓：优先观察 A 股一侧。
- 专业投资者：若能合法做空 H 股，可考虑多 A 空 H 的价差组合。

### 5.3 均值回归观察

价差不只看当天绝对值，也要看历史位置：

- 当前价差是否偏离 60/120 日均值。
- 当前价差在近 60/120 日分位数中的位置。
- 公司是否有短期事件导致两地预期不同。
- 两地成交量和流动性是否支持交易。

项目已有 `v_official_ah_premium_trend`，LLM 可以用它计算历史均值、分位和偏离。

## 7. LLM 如何推荐可观察标的

项目不会让 LLM 直接查询原始数据库，而是通过只读视图和 SQL Guard 控制范围。

A/H 套利相关视图：

- `v_latest_hk_connect_official_ah_premium`：最新交易日且港股通可操作的官方 AH/H/A 溢价。
- `v_official_ah_premium_trend`：官方 AH/H/A 溢价历史趋势。
- `v_watchlist_opportunity`：自选股阈值、距离和机会状态。
- `v_sync_health`：同步健康状态。

LLM 推荐逻辑应当遵守：

1. 先筛港股通可操作标的，避免推荐不可交易股票。
2. 区分方向：`AH 溢价高` 代表 H 股便宜；`AH 溢价为负` 代表 A 股便宜。
3. 同时检查 60/120 日均值或分位，不只看当天极值。
4. 优先推荐流动性较好、基本面可解释的标的；极端价差只列为高风险观察。
5. 输出必须给出明确结论、阈值、触发条件和反证条件。

## 8. 推荐提示词示例

用户可以问：

```text
请从最新港股通 AH 比价中，找 AH 溢价大于 20%、H 股相对便宜、且不是极端小票的前 10 只股票，并说明为什么只是观察机会不是无风险套利。
```

LLM 应生成类似 SQL：

```sql
SELECT
  a_ts_code,
  a_name,
  hk_ts_code,
  hk_name,
  ah_premium_pct,
  ha_premium_pct,
  connect_channels
FROM v_latest_hk_connect_official_ah_premium
WHERE ah_premium_pct > 20
ORDER BY ah_premium_pct DESC
LIMIT 10;
```

用户也可以问：

```text
找 A 股相对 H 股便宜的 AH 公司，按 AH 溢价从低到高排列，给我 8 个适合继续研究的例子。
```

对应 SQL：

```sql
SELECT
  a_ts_code,
  a_name,
  hk_ts_code,
  hk_name,
  ah_premium_pct,
  ha_premium_pct
FROM v_latest_hk_connect_official_ah_premium
WHERE ah_premium_pct < 0
ORDER BY ah_premium_pct ASC
LIMIT 8;
```

LLM 输出给用户时建议采用这种结构：

```text
以下是基于最新官方 AH 比价和港股通可操作范围筛选出的价差观察标的。

1. 中国平安 / 02318.HK
   - 当前 AH 溢价约 8.08%，H 股相对 A 股便宜。
   - 属于金融蓝筹，流动性较好，适合做跨市场替代观察。
   - 风险：保险行业估值受利率、权益市场和负债端假设影响。

2. 招商银行 / 03968.HK
   - 当前 AH 溢价约 -6.81%，A 股相对 H 股便宜。
   - 银行蓝筹，价差温和，适合比较 A/H 两地持有成本。
   - 风险：银行资产质量、息差和红利政策会影响估值。
```

## 10. 风险清单

- 价差可能长期不收敛。
- 不能把 A 股和 H 股简单视为可互换资产。
- 汇率、交易税费、印花税、融资融券成本可能吃掉价差。
- 两地交易日和交易时段不同，可能出现隔夜风险。
- 港股流动性和价差可能远弱于 A 股。
- 卖空和融券受券源、成本、规则限制。
- 极端 AH 溢价往往来自流动性、公司治理、行业风险或投机情绪，不一定是机会。

## 11. 后续产品增强建议

- 给每只 AH 股票计算 20/60/120 日 AH 溢价均值、标准差和分位数。
- 增加“价差偏离度”字段：当前溢价减 60 日均值。
- 加入成交额和换手率过滤，降低低流动性误报。
- 给 LLM 增加“保守/中性/进取”三种推荐模式。
- 与 `stock_selection_factor_snapshot` 联动，只推荐基本面质量较好的 AH 标的。
