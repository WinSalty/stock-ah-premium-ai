# A/H 溢价套现评审与展示设计

文档日期：2026-05-04

## 0. 实施状态

截至 2026-05-04，除“2.2 暂缓处理”和 “P4 实时能力回归”外，本文档中的官方主口径闭环、港股通可操作性、自选股能力和决策指标增强已完成代码落地。

已落地内容：

- LLM 只读视图已切换为官方 AH 比价主口径，并保留旧视图名作为兼容入口。
- SQL Guard 和 LLM schema 已开放官方 AH 趋势、最新官方 AH、港股通官方 AH 和自选机会视图。
- `/api/ah-premiums` 已返回港股通可操作性、`connect_channels`、自选状态、关注方向、阈值距离、60 日分位和 20/60/120 日均值。
- 新增 `watchlist_stock` 表、Alembic 迁移、后端 CRUD API 和前端自选入口。
- 首页已从“数据总览”调整为“自选机会台”，默认展示自选股机会卡片、阈值状态、通道、趋势线和均值线。
- 溢价页已支持只看港股通、只看自选、通道、A/H 与 H/A 区间筛选，并支持加入自选时维护目标阈值、编辑自选配置和取消自选。
- 智能问答页示例问题和请求上下文已围绕官方口径和自选股调整。
- “刷新实时”文案已调整为“重算派生”，避免在实时接口修复前误导为真实实时行情。

## 1. 目标校准

本项目的核心目标是为通过 A/H 溢价进行套现或换仓决策提供数据参考。多余数据落表是为了后续扩展，不应稀释当前页面、查询和问答的主决策链路。

当前实现应坚持以下口径：

- 主展示口径以 `official_ah_comparison` 为准。
- A/H 比价和 A/H 溢价直接使用官方字段。
- H/A 比价由官方 A/H 比价反推，H/A 溢价由 H/A 比价计算。
- `ah_premium_daily` 自算表暂不作为主展示来源；仅保留为后续具备港股日线权限后重新启用自算、实时或校验口径的扩展表。
- 实时接口当前存在问题，暂不纳入本轮落地；页面和文档应避免把“重算派生字段”误导成真实实时行情。
- 用户更可能关注自己熟悉或持有的部分股票，因此产品第一优先级应从“全市场榜单”转向“自选股票机会判断”。

## 2. 评审结论

### 2.1 需优先处理

1. LLM 只读视图应切换到官方主口径。

   当前只读视图 `v_latest_ah_premium`、`v_ah_premium_trend` 仍读取 `ah_premium_daily`。在自算表不作为主口径的前提下，智能问答会和前端总览、溢价页产生口径不一致。应将 LLM 默认查询视图迁移到 `official_ah_comparison`，或新增明确命名的官方视图，例如 `v_latest_official_ah_premium`、`v_official_ah_premium_trend`，并让 SQL Guard 默认只暴露官方视图。

2. 溢价榜应体现港股通可操作性。

   `/api/ah-premiums` 当前直接读取官方 AH 比价表，未按 `hsgt_constituent` 过滤当前交易日港股通标的，也没有返回沪市港股通、深市港股通通道。用于套现决策时，股票是否在当日港股通范围内是硬条件。榜单应关联 `hsgt_constituent`，返回 `is_hk_connect`、`connect_channels`，并支持按通道筛选。

3. 第一屏应优先展示用户关注股票。

   当前总览默认展示招商银行，并提供全局配对选择。对于真实使用场景，应把自选股作为默认入口：用户打开页面先看到关注股票是否达到阈值、是否接近机会、数据是否新鲜、是否可通过港股通操作。

### 2.2 暂缓处理

1. 实时刷新能力暂缓。

   当前“刷新实时”本质是重算官方表中的 H/A 派生字段，并设置实时标记。由于实时接口暂时存在问题，后续可以先改名为“重算派生指标”，等真实实时链路可用后再恢复“实时刷新”语义。

2. AH 配对生效日覆盖问题暂缓。

   `ah_stock_pair.effective_start_date` 在官方历史同步时可能被较新的交易日覆盖。由于自算表暂不作为主展示来源，该问题不阻塞当前主链路。若后续重新启用自算历史回算，应先修复为保留最早出现日期，或从官方配对快照中独立维护历史有效区间。

## 3. 推荐展示形态

### 3.1 首页改为“自选机会台”

首页第一屏建议由三部分组成：

1. 自选股机会卡片

   每张卡片展示一个关注标的，重点字段包括：

   - 股票名称和 A/H 代码。
   - 当前 A/H 溢价、H/A 溢价。
   - 用户关注方向，例如持 A 股看 H 股折价、持 H 股看 A 股溢价。
   - 用户阈值和当前距离阈值。
   - 近 20/60/120 个交易日历史分位。
   - 较 60 日均值偏离。
   - 港股通通道：`SH_HK`、`SZ_HK` 或两者。
   - 数据日期、来源和更新时间。

2. 机会分组

   按行动价值将自选股分组：

   - 已达阈值：当前溢价或折价已进入用户设置区间。
   - 接近阈值：距离阈值较近，需要持续观察。
   - 正常观察：未触发阈值。
   - 数据异常：缺官方比价、缺港股通名单、日期不一致或来源过旧。
   - 不可操作：不在当日港股通范围内。

3. 自选股趋势图

   趋势图默认跟随当前选中的自选股，显示：

   - A/H 或 H/A 溢价曲线。
   - 用户阈值线。
   - 20/60/120 日均值线。
   - 历史分位参考带。
   - 官方数据点和实时/手工重算点的来源标记。

### 3.2 溢价页改为“全市场筛选器”

溢价页继续保留全市场能力，但定位为从市场中发现候选股票。表格推荐列顺序：

1. 关注状态。
2. 股票名称、A 股代码、H 股代码。
3. 是否港股通、通道。
4. A/H 溢价、H/A 溢价。
5. 近 60 日分位。
6. 较 60 日均值偏离。
7. A 股收盘和涨跌幅。
8. H 股收盘和涨跌幅。
9. 数据日期、来源、更新时间。

筛选条件建议包括：

- 交易日。
- 关键词。
- 是否只看自选。
- 是否只看港股通可操作。
- 港股通通道。
- A/H 溢价区间。
- H/A 溢价区间。
- 历史分位区间。
- 数据来源和实时标记。

### 3.3 智能问答默认使用官方口径

问答页示例问题应围绕当前主口径和自选股：

- 我关注的股票里，最近一个交易日哪些 H/A 折价最明显？
- 哪些自选股已经达到我设置的阈值？
- 某只股票近 60 个交易日的 H/A 溢价处在什么历史分位？
- 最新交易日哪些官方 AH 比价数据不是港股通可操作标的？

LLM 回答必须说明口径：

- 数据来自 `official_ah_comparison`。
- H/A 为由官方 A/H 反推。
- 港股通可操作性来自 `hsgt_constituent`。
- 若查询使用自算表，必须显式说明自算表不是当前主展示口径。

## 4. 数据与接口建议

### 4.1 自选股表

建议新增 `watchlist_stock`：

| 字段 | 说明 |
| --- | --- |
| `id` | 自增主键 |
| `a_ts_code` | A 股代码 |
| `hk_ts_code` | H 股代码 |
| `display_name` | 用户自定义展示名 |
| `preferred_direction` | 关注方向，建议取值 `AH`、`HA` |
| `target_premium_pct` | 目标溢价或折价阈值 |
| `holding_market` | 当前持有侧，建议取值 `A`、`H`、`UNKNOWN` |
| `sort_order` | 展示排序 |
| `note` | 用户备注 |
| `is_active` | 是否启用 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 4.2 官方口径视图

建议新增或替换 LLM 只读视图：

- `v_latest_official_ah_premium`：最新交易日官方 AH/H/A 溢价。
- `v_official_ah_premium_trend`：官方 AH/H/A 溢价历史趋势。
- `v_latest_hk_connect_official_ah_premium`：最新交易日且属于港股通的官方溢价结果，包含 `connect_channels`。
- `v_watchlist_opportunity`：自选股机会判断视图，包含阈值、分位、通道、数据来源。

### 4.3 API 建议

- `GET /api/watchlist`：查询自选股和当前机会状态。
- `POST /api/watchlist`：新增自选股。
- `PATCH /api/watchlist/{id}`：更新阈值、方向、备注、排序。
- `DELETE /api/watchlist/{id}`：停用或删除自选股。
- `GET /api/ah-premiums`：增加 `only_hk_connect`、`only_watchlist`、`direction`、`min_ha_premium`、`max_ha_premium`、`channel`。
- `GET /api/ah-premiums/{a_ts_code}/{hk_ts_code}/trend`：返回均值线、分位信息和来源标记。

## 5. 实施优先级

### P0 文档与口径同步

已完成本评审与展示设计文档。后续所有实现应以官方主口径为默认口径。

### P1 官方口径闭环

1. 将 LLM 只读视图切换到 `official_ah_comparison`。
2. 更新 SQL Guard 白名单和 LLM schema 描述。
3. 在 `/api/ah-premiums` 关联港股通名单，返回 `is_hk_connect` 和 `connect_channels`。
4. 补充后端单元测试，验证官方视图、港股通筛选和通道返回。

### P2 自选股能力

1. 新增自选股表、迁移和基础 CRUD API。
2. 总览页默认展示自选股机会状态。
3. 溢价页支持只看自选、加入自选、编辑阈值和取消自选。
4. 智能问答支持按自选股范围查询。

### P3 决策指标增强

1. 增加 20/60/120 日均值和历史分位计算。
2. 趋势图增加阈值线、均值线和分位参考带。
3. 榜单增加“距离阈值”和“较均值偏离”。

### P4 实时能力回归

1. 修复真实实时接口。
2. 明确实时数据来源、刷新时间和覆盖范围。
3. 区分官方盘后数据、实时数据、手工重算派生指标。
4. 恢复或重命名“刷新实时”按钮。

## 6. 验收标准

- 用户打开首页即可看到自选股票当前是否有机会，而不是先从全市场榜单里搜索。
- 总览页、溢价页、数据查询页和智能问答默认口径一致，均以官方 AH 比价为主。
- 任一榜单结果都能看出是否港股通可操作，以及可用通道。
- LLM 回答明确说明官方口径、H/A 反推口径和港股通来源。
- 自算表不会在未明确说明的情况下参与主展示或默认问答。
- 实时能力未修复前，页面文案不暗示已经拉取真实实时行情。
